Modelowanie rwnowagi dobra i za w ukadach zoonych

Opis problemu

W badaniu ewolucyjnych ukadw spoecznych czsto analizuje si konkurencj pomidzy dwoma strategiami zachowania:

  • C dobro, wsppraca (cooperation),
  • D zo, egoizm (defection).

Niech:

  • ( x ) oznacza odsetek kooperatorw w populacji,
    przy czym $( 0 \leq x \leq 1 $),
  • ( 1 - x ) to odsetek defektorw.

Zadaniem rwnania replikatora jest opisanie, jak zmienia si udzia strategii C w czasie, w zalenoci od ich sukcesu (zysku) w interakcji z innymi.

redni zysk (payoff)

Kada strategia ma pewien redni zysk ((\pi)), ktry zaley od tego, z kim osobnik wchodzi w interakcj.
Dla dwch strategii (C) i (D) zapisujemy to nastpujco:

$$ \pi_C = R \cdot x + S \cdot (1-x) $$

$$ \pi_D = T \cdot x + P \cdot (1-x) $$

Gdzie:

Symbol Znaczenie Interpretacja
R Reward Nagroda za wspprac z innym kooperatorem (CC)
S Suckers payoff Strata lub saba korzy dla kooperatora, ktry spotyka defektora (CD)
T Temptation Pokusa defektora zysk przy spotkaniu kooperatora (DC)
P Punishment Kara zysk defektora przy spotkaniu innego defektora (DD)

redni zysk caej populacji

redni zysk wszystkich uczestnikw gry (zarwno C, jak i D) to warto oczekiwana z ich udziaw:

$$ \bar{\pi} = x \pi_C + (1-x) \pi_D $$

To wanie do tej wartoci odnosi si rwnanie replikatora strategie, ktre maj powyejredni zysk, bd si rozprzestrzenia w populacji.

Rwnanie replikatora

Zmienno odsetka kooperatorw w czasie opisana jest rwnaniem:

$$ \frac{dx}{dt} = x \, (\pi_C - \bar{\pi}) $$

czyli:

  • jeli $(\pi_C > \bar{\pi})$, wsppraca daje wicej ni rednia odsetek kooperatorw ronie,
  • jeli $(\pi_C < \bar{\pi})$, wsppraca si nie opaca udzia kooperatorw maleje,
  • jeli $(\pi_C = \bar{\pi})$, populacja osiga rwnowag ewolucyjn.

Opis badanych scenariuszy gier ewolucyjnych

W eksperymencie przeanalizowano kilka klasycznych modeli interakcji strategicznych, opisanych za pomoc czterech parametrw: ( R, S, T, P ) odpowiadajcych kolejno:

  • R (Reward) nagroda za wspprac z kooperatorem (CC),
  • S (Suckers payoff) strata kooperujcego, ktry trafi na defektora (CD),
  • T (Temptation) pokusa zdrady zysk defektora przy spotkaniu kooperatora (DC),
  • P (Punishment) kara zysk defektora przy spotkaniu innego defektora (DD).

Na ich podstawie mona sklasyfikowa rne gry spoeczne, opisujce zderzenie dobra i za (wsppracy i egoizmu).

1. Prisoners Dilemma (Dylemat winia)

Parametry: R = 3, S = 0, T = 5, P = 1

Interpretacja: Klasyczny dylemat wsppracy.
Zdrada (defekcja) daje jednostce wikszy zysk ni wsppraca, niezalenie od tego, co robi druga strona.

2. Snowdrift Game (Gra w odnieanie / Chicken Game)

Parametry: R = 3, S = 1, T = 4, P = 0

Interpretacja: Obie strony korzystaj, jeli kto wsppracuje, ale kady woli, eby to drugi ponis koszt.
Przykad: dwie osoby stoj przy zasypanym samochodzie lepiej, gdy kto odniey, ale niekoniecznie my sami.

3. Stag Hunt (Polowanie na jelenia)

Parametry: R = 4, S = 0, T = 3, P = 1

Interpretacja: Dwie osoby mog polowa na jelenia (wymaga wsppracy) lub na zajca (samodzielnie).
Jeli jedna zdradzi, druga traci dlatego wsppraca wymaga zaufania.

4. Cooperation Dominant (Dominacja dobra)

Parametry: R = 5, S = 2, T = 3, P = 1

Interpretacja: Wsppraca jest zawsze bardziej opacalna ni defekcja.
Kada interakcja przynosi wikszy zysk, gdy uczestnicy s kooperatorami.

5. Negative Payoffs (Ujemne wypaty)

Parametry: R = 2, S = -3, T = 1, P = -1

Interpretacja: Model konfliktowy, w ktrym wszystkie interakcje generuj straty (np. toksyczne rodowisko, konflikty spoeczne).
Mimo to, wsppraca moe ograniczy negatywne skutki.

Poniszy kod implementuje rwnanie replikatora dla piciu typw gier i symuluje zmiany odsetka kooperatorw w czasie.

import matplotlib.pyplot as plt
import numpy as np

parameter_sets = {
    "PrisonerDilemma": {"R": 3, "S": 0, "T": 5, "P": 1},
    "Snowdrift": {"R": 3, "S": 1, "T": 4, "P": 0},
    "StagHunt": {"R": 4, "S": 0, "T": 3, "P": 1},
    "CoopDominant": {"R": 5, "S": 2, "T": 3, "P": 1},
    "NegativePayoffs": {"R": 2, "S": -3, "T": 1, "P": -1},
}

x0 = 0.5
t_max = 10
dt = 0.001
times = np.arange(0, t_max, dt)

plt.figure(figsize=(10, 6))

for name, params in parameter_sets.items():
    R, S, T, P = params["R"], params["S"], params["T"], params["P"]

    def pi_C(x):
        return R * x + S * (1 - x)

    def pi_D(x):
        return T * x + P * (1 - x)

    def pi_bar(x):
        return x * pi_C(x) + (1 - x) * pi_D(x)

    def dx_dt(x):
        return x * (pi_C(x) - pi_bar(x))

    x_vals = np.zeros_like(times)
    x_vals[0] = x0

    for i in range(1, len(times)):
        x_vals[i] = x_vals[i - 1] + dx_dt(x_vals[i - 1]) * dt
        x_vals[i] = np.clip(x_vals[i], 0, 1)

    plt.plot(times, x_vals, label=name)

plt.xlabel("Czas t")
plt.ylabel("Odsetek kooperatorw x(t)")
plt.title("Ewolucja odsetka kooperatorw w rnych grach ewolucyjnych")
plt.grid(True)
plt.legend()
plt.show()
